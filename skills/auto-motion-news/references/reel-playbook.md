# the channel Reel Playbook

Use this playbook whenever a Reel, Shorts, MP4, motion card, or screen-recording edit is made for the channel or a similar Korean AI/tool/news channel.

The Reel must not feel like a card slideshow. It should feel like a fast visual proof: hook, show, explain, invite.

## Default Format

- Duration: 15-20 seconds for most AI/tool/news topics.
- Max duration: 30 seconds only when the topic is dense and the visual proof is strong.
- Format: 9:16 vertical.
- Safe zones: keep important text away from Instagram profile/caption area at the lower left and action buttons on the right.
- Font: match card-news typography. Use `Griun Mongtori` for Korean overlays unless the user explicitly approves another font.

## Retention Structure

1. 0-2s Hook
   - Start with motion, not a static title.
   - Show a real screen, typed prompt, command, cursor action, source zoom, before/after, or surprising result.
   - The first overlay should be understandable without context.

2. 2-8s What changed
   - One sentence explanation.
   - Use source proof: official UI, docs, repo, demo frame, screenshot, issue, pricing row, or product screen.

3. 8-14s How to use it
   - Show a practical action: install, search, ask, drag, approve, run, compare, upload, test, or check.
   - Prefer typing and zoom/callout over abstract animation.

4. 14-20s CTA
   - Ask for save/comment/follow.
   - When appropriate: "댓글에 키워드 남기면 링크/체크리스트 정리해드릴게욤."

## Motion Jobs

Every source-based Reel should include at least three meaningful motion jobs:

- typed prompt, command, search query, or chat reply;
- source screenshot zoom or pan;
- cursor/tap pulse;
- callout highlight over a real UI/detail;
- before/after split;
- checklist reveal;
- progress bar, timeline, or status change;
- phone/mockup scroll;
- terminal or side-panel typing;
- quick source proof flash with readable title/URL.

Do not count simple fade-in/fade-out as a motion job.

## Source And Visual Rules

Use real or topic-specific visuals by default.

Preferred source order:

1. official demo/video frame;
2. official docs/product/repo screenshot;
3. creator demo or community screenshot with attribution/reference notes;
4. HTML-native reconstruction of the workflow;
5. generated editorial visual specific to the exact topic.

Generated visual rule:

- Do not use generic AI robot scenes as the main proof.
- For Claude/OpenAI/GitHub/Google/Figma/Canva/etc., use source screenshots, UI reconstructions, or brand-specific colors and product context.
- Reject generated visuals with visible nonsense text.
- If the generated visual is only a mood image, pair it with real source proof in the same Reel.

## the channel Voice

Overlay copy should sound like a friendly Korean tutor, not a formal news anchor.

Good:

- "이거 그냥 발표가 아니라, 일하는 방식이 바뀌는 신호예요."
- "써보기 전에 이 부분은 꼭 체크해야 해요."
- "저장해두면 나중에 세팅할 때 바로 써먹습니다."

Avoid:

- "본 기능은 사용자 경험을 혁신합니다."
- "생산성 향상을 체감할 수 있습니다."
- "본 영상에서는 주요 내용을 정리합니다."

## Screen Recording Edit Mode

When the user gives raw screen footage:

- Cut dead air first.
- Preserve the real screen as the hero.
- Add motion graphics only to explain, not to cover.
- Use zooms/callouts on active UI areas.
- Blur or cut secrets, emails, tokens, billing screens, private repo names, and sensitive file paths.
- If the user will add subtitles later, do not burn final subtitles unless requested.

## QA Before Final

Fail and revise if:

- the first 2 seconds are static;
- it is only PNG card slideshow pacing;
- no real source or practical example appears;
- overlays are hidden by Instagram UI danger zones;
- text is too small for phone viewing;
- the Reel does not answer "그래서 어떻게 쓰는데?";
- the CTA is missing or feels bolted on.
