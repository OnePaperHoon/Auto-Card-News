# Media Research and Rights

Use this reference before searching, using, recreating, or attributing images and videos for motion-news projects.

## Search Targets

Prefer sources that help viewers understand the exact tool, feature, product, or story:

- Official announcement pages
- Official demos
- Product documentation
- GitHub README media
- App store screenshots
- Official YouTube videos
- Credible creator walkthroughs
- User-provided screenshots or recordings
- Open-license media

Avoid generic stock visuals when the viewer needs to understand a real tool or workflow.

## Usage Classification

Classify every visual candidate:

```text
Direct use: Safe enough to use with attribution or user permission.
Recreate: Use as reference, then rebuild as HTML/CSS, screenshot-style mockup, diagram, or generated visual.
Reference only: Do not place in final output; use only to understand the topic.
```

If rights are unclear, default to `Recreate` or `Reference only`.

## Attribution

Record:

- source title
- source owner
- URL
- date accessed when useful
- use type: direct use, recreate, or reference only
- final caption attribution

For captions, prefer a short source line:

```text
Source · <original owner/source>
참고 링크 · <url>
```

If the channel curated or edited the content, use:

```text
Contents Editor · <channel/person>
```

Do not use wording that implies the channel created the original tool, repo, paper, app, or demo.

## Video References

Official demo clips and walkthroughs can be useful, but do not use them just because they look exciting. Use a video reference only if it helps:

- show what changed
- show how the tool works
- prove the result
- make a workflow easier to understand

Record the usable segment if a clip is directly used or recreated.

## Recreation Patterns

When direct use is risky, recreate the point with:

- browser frame mockups
- simplified UI panels
- code blocks
- before/after cards
- animated arrows or cursor paths
- screenshot-style diagrams
- generated visual metaphors only when the topic does not require exact proof
