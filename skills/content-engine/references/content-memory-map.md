## Reference Families The User Liked

### AI Trend / HERMES Editorial

- Dark editorial base.
- Big, readable Korean title.
- Real screenshot or source crop at top.
- Short explanatory copy below.
- Source line present but small.
- Looks like an AI/media briefing, not a PPT.

Use for:

- OpenAI, Anthropic, Google, GitHub, Claude, Codex, Figma, security, agent, and research topics.

### VibeVoice / Viral Tech Poster

- GmarketSans.
- Dark tech base with subtle grid or glow.
- White rounded cards/chips.
- Mint/pink/yellow accent.
- One concrete result per card.
- Real source images or product UI, not random decoration.

Use for:

- GitHub repos, AI tools, API/product updates, hands-on workflows, source-to-output demos.

### Viral Local/Food/Popup Poster References

- Strong large text, loud contrast, sticker-like emphasis.
- Useful only as energy, not as a literal clone for AI content.
- If used, simplify. Do not create messy fireworks, cramped outlines, or unreadable text.

Use for:

- Very short hooks, event-like announcements, "무료/오픈소스/지금 해보기" posts.

## Topic Buckets That Fit 채널

- AI coding agents: Codex, Claude Code, Cursor, Copilot, agent memory, goals, routines, appshots.
- Open-source tools: GitHub repos with clear workflow payoff, recent hype, strong README/demo.
- AI security and policy: agent permissions, prompt injection, billing surprises, CI/CD risk, browser automation guardrails.
- Creator/productivity tools: voice, image, video, design, docs, spreadsheets, workflows.
- Research and prompt/loop engineering: only when translated into a practical method or checklist.
- AX consulting bridge: "우리 회사/팀이 이걸 어떻게 써야 하나?" topics such as GEO, rate limits, AI governance, workflow automation, internal enablement.

## What Usually Performs Better

- "이거 몰라서 손해 보는 상황" over "새 기능 발표".
- "직접 해볼 수 있는 1단계" over "개념 설명".
- "저장할 체크리스트" over "뉴스 요약".
- "댓글에 키워드 남기면 링크/정리" over vague follow CTAs.
- "실제 화면 + 한 줄 해석" over text-only cards.

## What Usually Performs Worse

- Dense official-summary cards.
- Acronym-first explanations.
- Same GitHub screenshot repeated.
- Abstract GPT-generated images.
- Tiny body text.
- Overly cute copy on security, billing, or serious developer-risk topics.
- No monetization/AX bridge.
